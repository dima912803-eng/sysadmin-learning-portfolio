# First Server Access

## Objective

Gain access to a production Linux environment and begin understanding how a real SaaS application is structured.

## Environment

Observed technologies:

- Linux
- SSH
- Node.js
- PM2
- Nginx
- DigitalOcean

## Activities Performed

- Connected to server using SSH
- Explored running services
- Reviewed PM2 processes
- Investigated application structure
- Discussed deployment process
- Started learning application architecture

## Key Questions

- How are deployments performed?
- How are backups managed?
- How many servers exist and what are their roles?
- How does offline synchronization work?

## Initial Findings

The application appears to use:

1. Linux operating system
2. Nginx reverse proxy
3. PM2 process manager
4. Node.js backend application

## Lessons Learned

A production application consists of multiple layers.

Understanding relationships between those layers is more important than simply memorizing commands.

## Next Steps

- Learn PM2 in detail
- Learn Nginx in detail
- Review backup strategy
- Understand synchronization architecture
